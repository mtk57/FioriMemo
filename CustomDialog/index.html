SAP Fiori ElementのFPM ExplorerでList Pageからカスタムダイアログを表示する手順
URL Flexible Programming Model Explorer
https://sapui5.hana.ondemand.com/test-resources/sap/fe/core/fpmExplorer/index.html
最終画面
My Custom Action をクリックすると
テーブル2つと閉じるボタンがあるカスタムダイアログが表示される。
修正前ソースコード
Page.controller.js
sap.ui.define(
    ["sap/fe/core/PageController", "sap/m/MessageBox", "sap/ui/model/Filter", "sap/ui/model/FilterOperator", "sap/ui/model/Sorter"],
    function (PageController, MessageBox, Filter, FilterOperator, Sorter) {
        "use strict";
        return PageController.extend("sap.fe.core.fpmExplorer.tableCustoms.Page", {
            onInit: function () {
                PageController.prototype.onInit.apply(this);
                this.getView().getModel("ui").setProperty("/isEditable", true);
                this.getView().getModel("ui").setProperty("/myColumnVisible", true);
            },
            hideMyColumn() {
                var oTable = this.byId("LineItemTablePageCustomColumns");
                // Warning: this API is still experimental
                oTable.hideColumns(["FirstColumnKey"]);
                this.getView().getModel("ui").setProperty("/myColumnVisible", false);
            },
            showMyColumn() {
                var oTable = this.byId("LineItemTablePageCustomColumns");
                // Warning: this API is still experimental
                oTable.showColumns(["FirstColumnKey"]);
                this.getView().getModel("ui").setProperty("/myColumnVisible", true);
            },
            onRowPressed(oEvent) {
                MessageBox.show("You pressed the row, you should navigate");
            },
            onPressAction(oEvent) {
                var oTable = this.byId("LineItemTablePageCustomActions");
                var aSelection = oTable.getSelectedContexts();
                MessageBox.show("You pressed the custom action with " + aSelection.length + " rows selected");
            },
            tableRefreshed(event) {
                var collectionBindingInfoAPI = event.getParameter("collectionBindingInfo");
                //Add a filter
                var filter = new Filter({
                    path: "BooleanProperty",
                    operator: FilterOperator.EQ,
                    value1: false
                });
                collectionBindingInfoAPI.addFilter(filter);
                //Add a sorter
                var sorter = new Sorter("ID", true);
                collectionBindingInfoAPI.addSorter(sorter);
                //Request an additional property to the request
                collectionBindingInfoAPI.addSelect(["TagStatus"]);
            },
            onPressInColumn(oEvent) {
                MessageBox.show("You pressed the custom action in the table");
            },
            onTableSelectionChanged(oEvent) {
                var oTable = this.byId("LineItemTableRowPress");
                MessageBox.show("Selection has changed. " + oTable.getSelectedContexts().length + " rows are selected.");
            },
            onPressMenuAction(oEvent) {
                MessageBox.show("You pressed the custom action: " + oEvent.oSource.mProperties.text);
            }
        });
    }
);

手順
1.FPM Explorerにアクセスして、Building Blocks - Table - Custom Colums / Actions を選択する。
　※基本的はどんなElementでも同じなはず。
2.Page.view.xml の15行目「Panel headerText」が"Table with Custom Actions"であることを確認して、zipファイルをダウンロードする。
3.ダウンロードされた「Table-CustomColumns_Actions.files.zip」を解凍すると、以下のファイルが展開される。
Table-CustomColumns_Actions.files
├ topics
│ └ buildingBlocks
│    └ table
│       └ tableCustoms これはFPM Explorer内でのサンプルの置き場所を示すフォルダ階層です。特に気にする必要はありません。この中にあるファイルが本体です。
│          ├ localService これはローカルの模擬（モック）サーバーのフォルダです。実際のバックエンドシステムに接続しなくても、この中にあるデータを使ってテーブルを表示・操作できるようにするためのものです。
│          │ ├ RootEntity.js データに関する追加のロジックが書かれている場合があります。
│          │ ├ RootEntity.json テーブルに表示される具体的なデータがJSON形式で入っています。
│          │ └ service.cds テーブルがどのようなデータ構造（項目名、データ型など）を持つかを定義するサービス定義ファイルです。
│          ├ manifest.json このUI5アプリケーションの心臓部です。アプリケーション全体の設定ファイルで、どのビューを最初に表示するか、どのデータソース（今回はlocalService）を使うか、などの重要な情報が定義されています。
│          ├ Page.controller.js 見た目を定義するファイル（View）です。テーブル、ボタンなどのUI部品がどのように配置されるかがXML形式で書かれています。FPM Explorerで編集していた画面の左側（エディタ部分）のXMLコードがこれです。
│          └ Page.view.xml 動きを定義するファイル（Controller）です。ボタンがクリックされたときに何をするか、といったアプリケーションのロジックがJavaScriptで書かれています。同じくエディタ部分のJavaScriptコードがこれにあたります。
└ sampleInfo.json FPM Explorerがこのサンプルを管理するための設定ファイルです。
4.Page.controller.jsを修正後ソースコードに従って修正する。
5.Table-CustomColumns_Actions.filesフォルダをzip圧縮する。
6.Table-CustomColumns_Actions.files.zipファイルをアップロードすると、FPM Explorerに反映される。
